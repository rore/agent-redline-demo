# Contributing to agent-redline-demo

This repo is a **paired demo** for [agent-redline](https://github.com/rore/agent-redline). Its content is regenerated by `scripts/sync-demo.sh` in the agent-redline repo from `agent-redline:demo-source/` and `agent-redline:examples/spring-hexagonal/`.

## You probably want to contribute to agent-redline, not here

If you want to:

- **Change the framework** (skill text, schema, reporter, extension shape) — open a PR against [agent-redline](https://github.com/rore/agent-redline). See its [CONTRIBUTING.md](https://github.com/rore/agent-redline/blob/main/CONTRIBUTING.md).
- **Change the demo's policy, scaffolded files, or scenarios** — open a PR against `demo-source/` in [agent-redline](https://github.com/rore/agent-redline). The maintainer will run `scripts/sync-demo.sh` to push the result here.
- **Fix a bug in the Spring fixture itself** (the `examples/spring-hexagonal/` source the demo is built from) — same: open a PR in [agent-redline](https://github.com/rore/agent-redline) against `examples/spring-hexagonal/`.

## What's appropriate to PR directly here

- **Fixing a typo in a generated file**, if you want a faster turnaround. The maintainer will eventually overwrite it with a sync, so this is a hotfix path. Not required.
- **Adding an issue** that's specific to this repo (e.g., the demo's CI workflow misbehaved, a specific PR scenario produced an unexpected verdict).

## Code of Conduct

Same as [agent-redline](https://github.com/rore/agent-redline). See its [CODE_OF_CONDUCT.md](https://github.com/rore/agent-redline/blob/main/CODE_OF_CONDUCT.md).
